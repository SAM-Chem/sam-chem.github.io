Prerequisites

SAM–Chem has the same build requirements as SAM v6.10.9 plus KPP 3.3.0:

  • A Fortran compiler (Intel ifort or GNU gfortran ≥ 9)
  • MPI implementation (OpenMPI, MPICH, or Intel MPI)
  • NetCDF-Fortran library
  • KPP 3.3.0 (for regenerating the chemistry code; not required to run)
  • Python 3 with numpy, scipy, and netCDF4 (for post-processing)

On a typical HPC cluster these are available as environment modules. For example, on a system using Lmod: 

module load intel/oneapi mpi/intel netcdf
Note: You do not need KPP to run SAM–Chem. The pre-generated Fortran source for the CloudChem mechanism is included in the repository. KPP is only needed if you want to compile a different mechanism.

Installation

Clone the repository from GitHub:

git clone https://github.com/sam-chem/sam-chem.git
cd sam-chem

The repository layout is:

sam-chem/
├── SRC/            # Fortran source (SAM + chemistry module)
├── CHEMISTRY/      # KPP mechanism files and generated Fortran
│   └── CloudChem/  # CloudChem mechanism (.kpp, .f90)
├── CASES/          # Example case directories
│   ├── GoAmazon_2D_WetSeason/
│   ├── GoAmazon_2D_DrySeason/
│   └── GoAmazon_3D_WetSeason/
├── SCRIPTS/        # Build helpers and post-processing utilities
└── Build/          # Build output (created at compile time)

Building SAM–Chem

SAM–Chem uses the same Makefile system as SAM. Edit Build/Makefile to point to your compiler and NetCDF paths: 

# Build/Makefile — edit these lines
FC     = mpiifort           # or mpif90 for GNU
NETCDF = /path/to/netcdf

Then compile from the Build/ directory:

cd Build
make -j 8

A successful build produces the executable Build/SAM_ADV_CGLS_BN2_MICRO_M2005_RAD_RRTM4PBL. The long name encodes the compile-time scheme selections.

Common issue: If the build fails with NetCDF linking errors, double-check that nc-config --flibs and nc-config --prefix match the paths in your Makefile. You may need to set LD_LIBRARY_PATH if your NetCDF is installed in a non-standard location.

Running a 2D Simulation

The GoAmazon 2D wet season case is the fastest way to test a working SAM–Chem installation. The domain is 256 columns × 256 vertical levels with no third dimension, so a 1-day test runs in roughly 15–30 minutes on 16 cores.

1. Copy and inspect the case directory

cp -r CASES/GoAmazon_2D_WetSeason CASES/my_2D_test
cd CASES/my_2D_test
ls

Key files in a case directory:

FilePurpose
prmRun-control namelist (domain size, time step, physics switches)
lsf.txtLarge-scale forcing (3-hourly, 25 mb resolution)
snd.txtInitial sounding (T, q, u, v profiles)
chemistry.nmlChemistry namelist (soil moisture, emission tuning, deposition switches)
run.shSubmission script for your job scheduler

2. Configure the run

Open prm and confirm the key namelist parameters: 

&PARAMETERS
  nx = 256, ny = 1, nz = 256,
  dx = 66.6, dy = 66.6,
  nstop = 86400,          ! stop after 1 day (seconds)
  noutput = 3600,         ! write output every hour
  ...
&END

Open chemistry.nml and check the soil moisture initial value and emission flags are set for the wet season: 

&CHEMISTRY
  do_chemistry   = .true.,
  do_lightning   = .true.,
  do_soil_nox    = .true.,
  do_isoprene    = .true.,
  do_dry_dep     = .true.,
  do_wet_dep     = .true.,
  soil_moisture  = 0.86,   ! MERRA-2 wet season value
&END

3. Submit the job

Edit run.sh for your cluster's scheduler, then submit:

# SLURM example
sbatch run.sh

# PBS/Torque example
qsub run.sh

The run.sh examples in each case directory target 16 cores (1 node). Adjust #SBATCH --ntasks and mpirun -np together.

Running a 3D Simulation

The 3D wet season case adds the second horizontal dimension (128 × 128 × 256), enabling resolved convective cells and three-dimensional transport. Budget roughly 1.5 days wall-time per simulation day on 256 cores. 

cp -r CASES/GoAmazon_3D_WetSeason CASES/my_3D_test
cd CASES/my_3D_test

The prm file in this case sets nx = 128, ny = 128. Everything else is identical to the 2D case. The run.sh example requests 256 MPI tasks across 8 nodes (32 cores per node).

Tip: Run a short 2D test (1–3 days) first to verify your build and forcing files are correct before committing to a multi-day 3D run.

Output Files

SAM–Chem writes NetCDF output in several categories:

File patternContentsDefault frequency
*.nc2d 2D surface and column fields (precipitation, SWdn, LWdn, LNOx flash rate) Hourly
*.nc3d 3D instantaneous snapshots (T, q, u, v, cloud water, all tracers) 3-hourly
*.ncstat Domain-averaged vertical profiles and statistics 10 minutes
*.ncchem Domain-averaged chemistry profiles and column integrals 10 minutes
*.restart Binary restart file (all prognostic variables) Daily

Output frequencies are controlled in prm:

noutput  = 3600    ! 3D output interval (seconds)
nstat    = 600     ! statistics interval (seconds)
ncheckpoint = 86400 ! restart write interval (seconds)

Basic Analysis

The SCRIPTS/ directory includes Python utilities for reading SAM–Chem output. Open the statistics file: 

import netCDF4 as nc
import numpy as np
import matplotlib.pyplot as plt

ds = nc.Dataset('GoAmazon_2D_WetSeason_STATS.ncstat')
t  = ds.variables['time'][:]        # hours
z  = ds.variables['z'][:]           # km
ISOP = ds.variables['ISOP'][:]      # ppb, (time, z)

plt.figure()
plt.pcolormesh(t, z, ISOP.T, shading='auto', cmap='YlGn')
plt.colorbar(label='Isoprene (ppb)')
plt.xlabel('Time (h)'); plt.ylabel('Height (km)')
plt.title('Domain-mean isoprene')
plt.savefig('isoprene_profile.png', dpi=150)

Chemical species in .ncstat and .ncchem files follow the CloudChem naming convention: ISOP, OH, HO2, O3, NO, NO2, HNO3, IEPOX, ISOPOOH, ISOP1Nit, ISOPDiNit, PAN, etc. All concentrations are in ppb.

Next Steps

  • Swap the chemistry mechanism — compile any KPP-compatible .kpp file with KPP 3.3.0, place the output in CHEMISTRY/<YourMechanism>/, and recompile. See CHEMISTRY/README.md for instructions.
  • Change the large-scale forcing — replace lsf.txt and snd.txt with ARM-format forcing files for a different campaign. SAM's standard tools for constructing these files work unchanged in SAM–Chem.
  • Extend emissions — additional surface emission fluxes can be added as new tracers via the SURF_FLUX interface in SRC/chemistry_surface.f90.
  • Compare with GEOS–Chem — scripts for regridding GEOS–Chem Classic output to SAM–Chem domain coordinates are included in SCRIPTS/geos-chem-comparison/.
